# 📜 Руководство и правила по оформлению HTML для AA DNA

Данный документ содержит правила и рекомендации по верстке и вставке HTML кода в статьи и публикации ДНК-проекта **AA DNA** через локальную CMS админку (`aadna-local`).

---

## 🤖 1. Автоматическая обработка в CMS (Кнопка «Вставить HTML»)

При использовании кнопки **«🌐 Вставить HTML»** в CMS админке происходит автоматическая очистка кода:
* **Удаление служебных тегов**: автоматически вырезаются теги `<!DOCTYPE>`, `<html>`, `<head>`, `<title>`, `<meta>`, `<body>`, `</body>` и `</html>`.
* **Нормализация отступов**: убираются 4-пробельные начальные отступы перед HTML-тегами, чтобы парсер Markdown (Zola) не превращал теги в блоки кода `<pre><code>`.
* **Адаптация стилей под темы**: в стили `.jir-history-wrapper` подставляются CSS-переменные и медиа-запросы `[data-theme="dark"]` / `[data-theme="light"]`.
* **Замена тире**: длинные тире `—` и `–` автоматически заменяются на обычные дефисы `-` или двоеточия `:`.

---

## ⚠️ 2. Обязательные требования к верстке

### 1. Изоляция CSS классов (Критически важно!)
* **Запрещено переопределять базовые теги глобально**: не пишите в теге `<style>` правила вида `p { margin: 20px; }` или `h1 { color: red; }`, иначе эти стили применятся ко всему сайту (шапке, меню, подвалу).
* **Всегда используйте родительский селектор**:
  ```html
  <!-- ПРАВИЛЬНО -->
  <style>
    .my-custom-card {
      max-width: 800px;
      margin: 0 auto;
    }
    .my-custom-card p {
      line-height: 1.8;
      margin-bottom: 1rem;
    }
    .my-custom-card .title {
      font-size: 1.5rem;
      font-weight: 700;
    }
  </style>

  <div class="my-custom-card">
    <h2 class="title">Заголовок статьи</h2>
    <p>Текст публикации...</p>
  </div>
  ```

---

### 2. Валидность и парность тегов
* Каждый открытый тег (`<div>`, `<p>`, `<span>`, `<style>`, `<ul>`, `<li>`) должен иметь закрывающий парный тег (`</div>`, `</p>`, `</span>`, `</style>`, `</ul>`, `</li>`).
* Незакрытый тег `<div>` ломает разметку всей страницы, заставляя последующие блоки сайта рендериться внутри вашего элемента.

---

### 3. Адаптивность для мобильных устройств
* Избегайте фиксированной ширины в пикселях (например, `width: 1200px` или `width: 900px`).
* Используйте `max-width` и процентную ширину `width: 100%`:
  ```css
  .my-custom-card {
    max-width: 800px;
    width: 100%;
    margin: 2rem auto;
    padding: 1.5rem;
  }
  
  @media (max-width: 600px) {
    .my-custom-card {
      padding: 1rem;
    }
  }
  ```

---

### 4. Поддержка тёмной и светлой тем (Dark / Light Mode)
* Не используйте жёстко заданные чёрные или белые фоны без учета темы оформления.
* Рекомендуется использовать полупрозрачные фоны или CSS-переменные:
  ```css
  /* Использование полупрозрачного фона, подходящего под обе темы */
  .my-custom-card {
    background-color: rgba(255, 255, 255, 0.04);
    border: 1px solid rgba(255, 255, 255, 0.12);
    border-radius: 12px;
  }

  /* Или явная адаптация под темы */
  [data-theme="dark"] .my-custom-card {
    background: rgba(15, 22, 42, 0.6);
    color: #e2e8f0;
  }

  [data-theme="light"] .my-custom-card {
    background: #fcfbfa;
    color: #3b332c;
  }
  ```

---

### 5. Изображения и медиафайлы
* Теги `<img>` должны содержать HTTPS ссылки на изображения или относительные пути к медиапапкам проекта (например, `/media/legacy/...` или `/media/results/...`).
* Добавляйте `max-width: 100%` и `height: auto` к изображениям внутри блоков, чтобы они не выходили за пределы экрана.

---

## 💡 Чек-лист перед сохранением статьи

1. Откройте CMS `aadna-local`.
2. Нажмите кнопку **«🌐 Вставить HTML»** над полем ввода.
3. Вставьте ваш HTML код и нажмите **«Применить»**.
4. Воспользуйтесь кнопкой **«👁️ Предпросмотр»** для валидации внешнего вида публикации.
